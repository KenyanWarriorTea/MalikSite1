# Локальная сетевая конфигурация

Для работы MalikSite на локальной сети школы, выполните следующие шаги:

## 1. Подготовка учительского компьютера

### Установка требуемого ПО
- Docker Desktop (https://www.docker.com/products/docker-desktop)
- Git (если нужно клонировать проект)

### Получение IP адреса компьютера

#### На Windows (PowerShell):
```powershell
ipconfig
```
Найдите строку "IPv4 Address" под вашим активным сетевым адаптером (например: 192.168.1.100)

#### На Linux/Mac (Terminal):
```bash
ifconfig
# или
hostname -I
```

### Установка переменных окружения

Создайте файл `.env` в корневой папке проекта:

```bash
# .env файл
TEACHER_IP=192.168.1.100  # Замените на вашу локальную IP адреса компьютера
EXTERNAL_PORT=8000        # Порт для доступа со студенческих компьютеров
JUDGE0_PORT=2358

# Доступы (измените для безопасности)
TEACHER_CODE=teacher123
STUDENT_CODE=student123

# База данных
POSTGRES_USER=malik_user
POSTGRES_PASSWORD=malik_secure_password_123
```

## 2. Запуск приложения

### Вариант A: Docker Compose (рекомендуется)

```bash
cd /путь/к/MalikSite1
docker-compose up --build
```

Приложение будет доступно по адресу:
- **Локально**: http://localhost:8000
- **По сети**: http://192.168.1.100:8000 (замените на вашу IP адреса)

### Вариант B: Запуск на хосте (без контейнеризации)

```bash
cd backend
pip install -r requirements.txt
export DATABASE_URL=postgresql://user:password@localhost:5432/tasks_db
export JUDGE0_URL=http://localhost:2358
uvicorn app:app --host 0.0.0.0 --port 8000
```

## 3. Подключение студентов

### Со студенческих компьютеров

Откройте браузер и перейдите по адресу:
```
http://[IP_УЧИТЕЛЯ]:8000
```

Например, если IP учителя 192.168.1.100:
```
http://192.168.1.100:8000
```

Введите:
- Имя студента
- Код доступа студента (по умолчанию: `student123`)

## 4. Проверка подключения

### Команда для проверки доступности сервера

```bash
# С любого компьютера в сети:
curl http://192.168.1.100:8000

# Если доступно, вернет HTML код главной страницы
```

### Отладка проблем подключения

**Проблема: "Соединение отказано" (Connection refused)**
- Убедитесь, что контейнеры запущены: `docker ps`
- Проверьте, что портят не заблокированы: `netstat -an | grep 8000`
- Выключите брандмауэр (или добавьте исключение для порта 8000)

**Проблема: "Хост недоступен" (Host unreachable)**
- Убедитесь, что учителю и студентам в одной сети
- Проверьте IP адрес: используйте действительный локальный адрес (192.168.x.x, 10.x.x.x или 172.16.x.x)

**Проблема: Judge0 не оценивает код**
- Убедитесь, что контейнер Judge0 запущен: `docker ps | grep judge0`
- Проверьте логи: `docker logs [judge0_container_id]`

## 5. Мониторинг и логи

### Просмотр логов приложения

```bash
# Все сервисы
docker-compose logs -f

# Только веб-приложение
docker-compose logs -f web

# Только Judge0
docker-compose logs -f judge0

# Только база данных
docker-compose logs -f db
```

### Проверка статуса контейнеров

```bash
docker-compose ps
```

## 6. Закрытие приложения

```bash
# Остановить контейнеры
docker-compose stop

# Остановить и удалить контейнеры
docker-compose down

# Остановить и удалить данные (внимание: удалит базу данных!)
docker-compose down -v
```

## 7. Безопасность

- Измените коды доступа (TEACHER_CODE, STUDENT_CODE) перед использованием в реальных условиях
- Используйте пароль базы данных вместо стандартного
- Ограничьте доступ через брандмауэр школы
- Используйте HTTPS в production (потребует SSL сертификата)

## 8. Справочник портов

| Сервис | Порт | Назначение |
|--------|------|-----------|
| Web App (FastAPI) | 8000 | Основное приложение |
| Judge0 API | 2358 | Оценка кода |
| PostgreSQL | 5432 | База данных (только локально) |
| Redis (Judge0 DB) | 6379 | Кэш Judge0 (только локально) |
